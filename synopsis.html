1. Introduction

The rapid growth of digital commerce has transformed the way businesses operate and consumers shop. However, a large number of existing e-commerce platforms — especially those built for small and medium businesses — suffer from critical shortcomings including weak authentication mechanisms, poor scalability under high traffic, insecure payment handling, and the absence of intelligent user assistance features.

Common problems observed in existing systems include:

  • Lack of server-side input validation leading to SQL Injection and XSS vulnerabilities
  • Monolithic architectures that fail to scale under concurrent user load
  • Poorly designed database schemas causing performance bottlenecks
  • No real-time order tracking or admin visibility
  • Absence of smart assistance tools like chatbots for user guidance
  • Dependency on outdated or insecure payment flows

This project — Star King Power Tools E-Commerce Platform — is developed as a solution to these challenges. It is a fully functional, cloud-deployed, full-stack web application built for a real power tools business.

Key highlights of the system:

  • RESTful API backend built on Node.js and Express.js
  • Secure user authentication with password hashing
  • Razorpay payment gateway integration (test mode)
  • Rule-based JavaScript chatbot for customer assistance
  • Admin dashboard with real-time data management
  • Cloud deployment on Render with Aiven MySQL database
  • Responsive UI compatible across all devices

2. Main and Sub Points

A. Secure User Authentication System

  • User registration and login with SHA-256 password hashing
  • Session management using localStorage tokens
  • Protected routes for authenticated users only

B. Product Management System

  • Dynamic product catalog loaded from MySQL database
  • Category-based filtering and keyword search
  • Detailed product pages with image gallery and specifications

C. Cart & Order Processing

  • Persistent shopping cart using browser localStorage
  • GST calculation (18%) applied automatically
  • Multi-step checkout with address and delivery options
  • Order confirmation with unique order ID generation

D. Payment Gateway Integration

  • Razorpay integration supporting UPI, Credit/Debit Card, Net Banking
  • Server-side payment signature verification using HMAC-SHA256
  • Cash on Delivery as fallback option
  • Secure order saving post payment verification

E. Admin Dashboard

  • Separate admin panel with credential-based access
  • Real-time customer, order, product, and transaction management
  • CRUD operations on product catalog
  • Order status update functionality

F. Chatbot System

  • Rule-based chatbot built in vanilla JavaScript
  • Handles FAQs, product queries, order help, and contact info
  • No external API dependency — fully client-side

G. Performance & Scalability Features

  • MySQL connection pooling for concurrent request handling
  • Static file serving via Express for fast asset delivery
  • Cloud-hosted on Render with auto-deploy from GitHub
  • SSL-enabled Aiven MySQL database

3. Key Features (Detailed)

3.1 User Authentication & Security

  • Passwords stored as SHA-256 hashed strings — never in plain text
  • Input sanitization on both client and server side to prevent XSS attacks
  • Parameterized SQL queries using mysql2 prepared statements to prevent SQL Injection
  • User session stored securely in localStorage with structured JSON tokens

3.2 Product Browsing, Filtering & Search

  • Products loaded dynamically from MySQL via /api/products REST endpoint
  • Client-side filtering by category, price range, and keyword
  • Fallback to static products-data.json if API is unavailable
  • Product detail pages with multiple images, description, and add-to-cart

3.3 Shopping Cart System

  • Cart persisted in localStorage — survives page refresh
  • Real-time quantity update, item removal, and cart total recalculation
  • GST (18%) and optional express delivery charge (₹149) applied dynamically
  • Promo code system with percentage-based discounts

3.4 Razorpay Payment Integration

  • Backend creates Razorpay order via /api/payment/create-order
  • Frontend opens Razorpay checkout popup with prefilled user details
  • On success, payment signature verified server-side using HMAC-SHA256
  • Verified orders saved to MySQL with razorpay_order_id and razorpay_payment_id
  • COD orders bypass Razorpay and are saved directly

3.5 Order Tracking System

  • Orders stored in MySQL with status field (pending, paid, shipped, delivered)
  • Users can view order history on their profile page
  • Admin can update order status from the dashboard

3.6 Admin Panel (CRUD Operations)

  • Separate admin login with credential validation
  • Dashboard shows total orders, revenue, customers, and products
  • Full CRUD on product catalog — add, edit, delete products
  • View all customers, orders, and transaction records

3.7 Rule-Based Chatbot

  • Keyword-matching engine built in pure JavaScript
  • Responds to queries about products, pricing, delivery, returns, and contact
  • Floating widget accessible on all pages — no server calls, instant response

3.8 Security Measures

  • CORS configured on Express to restrict cross-origin requests
  • Environment variables used for all sensitive credentials (.env file)
  • .gitignore ensures secrets are never pushed to version control
  • SSL mode enabled on database connection
  • Razorpay webhook signature verification prevents payment tampering

3.9 Performance Optimization

  • MySQL connection pool (connectionLimit: 10) handles concurrent requests efficiently
  • Static assets served directly by Express without additional overhead
  • Optimized images for faster page load

3.10 Scalable Database Design

  • Four normalized tables: users, orders, products, contacts
  • JSON columns used for flexible storage of cart items and delivery addresses
  • BIGINT primary keys using Date.now() for distributed-safe ID generation
  • Tables auto-created on server startup via initDB() — zero manual setup

5. Challenges and Limitations

  • Payment Gateway in Test Mode: Razorpay is currently in test mode. Live transactions require business KYC verification.
  • Hosting Performance Constraints: Free tier on Render causes cold start delay of 30–50 seconds after inactivity.
  • Security Implementation Complexity: Managing HMAC-SHA256 verification and environment variables across local and cloud environments required careful configuration.
  • No AI-Based Recommendation System: The chatbot is rule-based and does not use machine learning. Product recommendations are not personalized.
  • Scalability Ceiling: Single-instance deployment does not support horizontal scaling without a paid plan or containerized deployment.
  • Image Storage: Product images are stored as static files. A cloud storage solution like AWS S3 would be needed for dynamic uploads.

7. Conclusion

The Star King Power Tools E-Commerce Platform is a real-world, cloud-deployed full-stack web application that demonstrates the complete software development lifecycle — from requirement analysis and database design to frontend development, backend API development, payment integration, and cloud deployment.

The system successfully addresses the core problems of existing e-commerce platforms by implementing secure authentication, parameterized database queries, server-side payment verification, and a scalable RESTful architecture. It is not a prototype — it is a live, functional system accessible on the internet.
